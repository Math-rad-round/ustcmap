# 考试卡理论

## 数据结构

```json
{
  "id": "test_...",
  "name": "",
  "content": "",
  "drawnum": 5,
  "requirement": 10,
  "difficulty": "normal",
  "achievementMode": "normal",
  "start": [],
  "effect": [],
  "result": []
}
```

- `drawnum`：考试阶段抽取课程牌数量。上学期仍按规则额外多抽 1 张。
- `requirement`：基础及格线。普通考试建议 10，简单考试建议 8，困难考试建议 15 或使用 1.5 倍难度。
- `difficulty`：建议取 `easy` / `normal` / `hard`，用于 UI 标签和后续成就档位。
- `achievementMode`：建议取 `easy` / `normal` / `hard`，简单考试给较少成就点，困难考试给困难级别成就点。
- `start`：每学期开始时，先抽取并选择考试卡，结算被选择考试的 `start`，然后进入事件阶段。
- `effect`：考试阶段中，在永久成果结算之后、课程效果之前结算。
- `result`：考试最后结算。适合“如果及格”“如果某符号达到阈值”这类结果奖励。

## 总体修正方向

- 考试分数回归 `10 / 15` 档次，普通考试基础及格线以 10 为中心。
- 所有考试吃符号的比重提高，让符号路线有更高上限。
- 允许每学期抽 2 张考试卡，选择 1 张。这样玩家可以根据当前符号、项目和成果结构选择考试，降低符号路线的不稳定。
- 可以加入更难但奖励更高的考试，也可以加入简单但成就点较少的考试。

## 新增考试设计

### 数学竞赛选拔

难度：1.5 倍 / 困难  
抽牌：5  
成就：获得困难级别的成就点。  
效果：

- 每张理论符号 +5 分。

设计目的：理论路线的高风险高回报考试。

### 公共表达展示

难度：普通  
抽牌：6  
效果：

- 每张社会符号 +1 分。
- 每张理论符号 +1 分。

设计目的：社会/理论混合考试，偏向表达、写作和逻辑组织。

### 校园影响力评估

难度：1.5 倍 / 困难  
抽牌：5  
效果：

- 每张社会符号 +2 分。
- 每个成果 +4 分。

结果：

- 如果本次考试及格，且社会符号 >= 4，额外获得 2 成就点。

设计目的：社会路线的困难高收益考试，奖励成果沉淀。

### 自学能力挑战

难度：普通  
抽牌：5  
开始效果：

- 本次考试理论符号 +3。
- 本次考试实践符号 +3。
- 本次考试社会符号 +3。
- 获得 3 温习值。

效果：

- 无。

设计目的：只能依靠课程和临时符号基础得分，不靠考试额外计分。需要引擎支持 start 中增加本次考试临时符号。

### 企业实践考核

难度：0.5 倍 / 简单  
抽牌：4  
成就：使用简单考试成就点档位。  
效果：

- 每个项目 +3 分。
- 每张事件卡 +2 分。

设计目的：项目铺设和事件手牌保留路线的简单考试。需要引擎支持 `event` 或 `eventHand` 作为计分变量。

## 需要引擎补充的效果/变量

- `getLowest`：取理论、实践、社会中最低符号数。
- `completedProject`：完成项目数量。
- `doublePermanentEffects`：本次考试永久成果效果翻倍。
- `randomExamSymbol`：考试选择后随机理论/实践/社会之一，输出为变量。
- `addTempSymbol`：只在本次考试中增加临时符号。
- `eventHand`：当前事件手牌数量，用于企业实践考核考试计分。
- `achievementMode` 或 `achievementTable`：支持简单/普通/困难考试的不同成就档位。
